# Tripline 서비스 기획서

작성일: 2026-04-26  
기준 버전: 현재 저장소 UI 프로토타입 최신안

## 1. 서비스 한 줄 정의

**Tripline은 사용자가 여행을 먼저 만들고, 그 여행 안에서 일정, 준비물, 날씨, 지출을 함께 운영하는 여행 실행 앱이다.**

## 2. 서비스가 해결하려는 문제

여행을 준비하고 다녀오는 과정에서 정보가 여러 곳으로 흩어진다.

- 장소는 지도 앱에 저장한다.
- 일정은 메모나 별도 여행 앱에 적는다.
- 준비물은 체크리스트 앱이나 종이에 적는다.
- 지출은 카드 앱이나 가계부 앱에서 따로 본다.
- 여행이 여러 개면 날짜별 소비 흐름을 한 번에 보기 어렵다.

Tripline은 이 흐름을 하나의 여행 단위로 묶는다.

- 먼저 여행을 만든다.
- 그 여행 안에서 day별 장소와 동선을 짠다.
- 같은 여행 맥락에서 준비물을 체크한다.
- 같은 여행의 날씨와 시차를 본다.
- 같은 여행의 지출을 기록한다.
- 전체 캘린더에서는 모든 여행의 지출을 날짜 기준으로 다시 모아본다.

즉 Tripline은 `여행 추천 앱`이 아니라 **여행을 실제로 운영하게 해주는 앱**이다.

## 3. 핵심 제품 철학

### 3-1. 여행이 가장 먼저다

Tripline의 모든 기능은 `여행` 단위로 움직인다.

- 여행이 없으면 먼저 여행을 만든다.
- 여행이 있으면 그 여행을 선택한 뒤 상세 기능을 쓴다.

### 3-2. 계획과 실행을 끊지 않는다

Tripline의 중심은 단순한 장소 저장이 아니다.

- 장소를 day별 일정으로 본다.
- 동선을 바꾼다.
- 메모를 끼워 넣는다.
- 실제 쓴 돈을 같은 여행 맥락에서 본다.

### 3-3. 화면은 가볍고 빠르게 읽혀야 한다

UI는 과한 카드와 장식성 배너보다 아래 원칙을 우선한다.

- 흰 배경 중심
- 회색 기반 구조
- 필요한 곳만 블루 포인트
- 플랫한 섹션 구조
- 바텀시트 중심 빠른 조작

## 4. 핵심 사용자

### 4-1. 1차 사용자

- 자유여행 일정을 직접 짜는 사용자
- 지도에 저장한 장소를 실제 일정으로 정리하고 싶은 사용자
- 여행 중 오늘 일정과 지출을 함께 보고 싶은 사용자

### 4-2. 2차 사용자

- 부모님, 가족과 함께 여행해서 동선과 준비물이 더 중요한 사용자
- 해외여행 중 날씨, 환율, 현지 시차를 함께 참고하고 싶은 사용자

## 5. 서비스 구조

### 5-1. 앱 구조의 핵심

Tripline의 중심은 `현재 선택된 여행`이다.

- 사용자는 여행 생성 직후 또는 보관함/검색 결과에서 여행을 탭할 때 `현재 선택된 여행`이 설정된다.
- 일정, 준비물, 날씨, 여행 전용 캘린더는 모두 이 `현재 선택된 여행`을 기준으로 동작한다.
- 여행이 없으면: 여행 생성 또는 보관함 진입
- 여행이 있으면: 그 여행 안에서 일정, 준비물, 날씨, 여행 전용 캘린더 사용
- 전역 캘린더는 예외적으로 모든 여행의 지출을 날짜 기준으로 모아보는 화면

### 5-2. 메인 탭 구조

- `홈`
- `보관함`
- `일정`
- `캘린더`

## 6. 탭별 역할

### 6-1. 홈

`현재 무엇을 해야 하는지` 보여주는 실행 대시보드

진행 중 여행이 있을 때:

- 현재 여행 요약
- 다음 일정
- 오늘 동선 요약
- 오늘 날씨
- 오늘 환율
- 오늘 지출 / 누적 지출
- 최근 기록

진행 중 여행이 없고 현재 선택된 여행만 있을 때:

- 선택된 여행 요약
- 일정 / 준비물 / 날씨 진입

여행이 아예 없을 때:

- 새 여행 만들기

홈은 여행 목록 전체를 보여주는 화면이 아니라, **오늘의 실행 정보**를 보여주는 화면이다.

### 6-2. 보관함

내가 만든 여행을 모아보는 허브

- 진행 예정 / 진행 중 / 완료 필터
- 여행 검색
- 작은 `+` 새 여행 만들기 액션
- 여행 선택

### 6-3. 일정

선택된 여행의 핵심 화면

- 상단 여행 정보
- 지도 미리보기
- day별 타임라인
- 장소 카드
- 항공편 카드
- 숙소 카드
- 거리 / 도보 / 차량 정보
- day별 `항공편 추가`
- day별 `숙소 추가`
- day별 `장소 추가`
- day별 `메모 추가`
- 여행 가계부 진입
- `편집` 진입

일정 탭에 여행이 없으면:

- 먼저 여행을 만들거나
- 보관함에서 여행을 선택하도록 안내한다.

### 6-4. 캘린더

모든 여행의 지출을 날짜 기준으로 모아보는 전역 화면

- 월간 캘린더
- 월 지출 합계
- 날짜별 지출 존재 여부
- 하단 상세 시트

하단 시트는 위로 올릴 수 있고, 위로 올리면:

- 월 전체 대신 선택된 `주간만` 보이고
- 아래 영역에는 상세 지출 내역이 크게 보인다.

## 7. 여행 안에서 사용하는 하위 기능

선택된 여행 안에서는 아래 기능을 사용한다.

- 일정 편집
- 장소 검색 / 추가
- 장소 상세
- 메모 시트
- 준비물 체크리스트
- 체크리스트 편집
- 아이템 추가
- 날씨
- 환율 참고
- 여행 전용 캘린더
- 지출 입력
- 상위 카테고리 관리
- OCR 가져오기 / 검수
- PDF 공유

## 8. 핵심 사용자 흐름

### 8-1. 여행 생성 흐름

1. 사용자가 홈 또는 보관함에서 `새 여행 만들기`로 진입한다.
2. `해외도시 / 국내도시` 탭과 지역 필터 또는 검색으로 여행 도시를 선택한다.
3. 여러 도시를 선택한 경우 하단 선택 영역에서 도시 순서를 확인하고, 필요하면 순서를 변경한다.
4. 여행 시작일과 종료일을 선택한다.
5. `누구와`, `여행 스타일`을 선택하거나 `다음에 하기`로 건너뛴다.
6. 저장하면 여행명은 첫 도시 기준으로 자동 생성한다.
   예: `후쿠오카, 도쿄, 오사카 여행`
7. 보관함 목록에 추가된다.
8. 생성한 여행이 `현재 선택된 여행`으로 자동 지정된다.
9. 생성 직후 일정 화면으로 이동하며, day별 `장소 추가 / 메모 추가`가 가능한 빈 일정 상태를 보여준다.

중요 정책:

- 예산은 핵심 입력에서 제외한다.
- Tripline은 예산 앱보다 **실제 지출 중심**으로 간다.
- 여행 생성 시 선택한 도시 목록과 순서는 여행의 목적지 목록으로 저장한다.
- 첫 번째 선택 도시는 대표 도시로 사용한다.
- `누구와`, `여행 스타일`은 추천/필터용 보조 정보이며 필수 입력은 아니다.
- 일정 화면에는 구매 관련 광고 배너, 셀프패키지, 일행과 함께 일정짜기 기능을 노출하지 않는다.

### 8-2. 일정 작성 흐름

1. 사용자가 여행을 선택한다.
2. 일정 화면으로 진입한다.
3. 항공편, 숙소, 장소, 메모 중 필요한 일정 항목을 추가한다.
4. 항공편 추가 시 출발일, 항공사/편명, 출발/도착 공항, 출발/도착 일시를 입력한다.
5. 숙소는 검색 후 체크인/체크아웃 날짜를 선택하고, 선택한 날짜 범위의 각 day에 숙소 카드로 반영한다.
6. 장소 검색으로 장소를 추가하고 day별로 배치한다.
7. 일정 화면의 `가계부` 버튼에서 현재 여행 전용 지출 화면으로 이동하고, day별 비용을 추가한다.
8. 드래그로 순서를 바꾸거나 다른 day로 이동한다.
9. 지도 미리보기를 탭하면 전면 동선 지도로 이동한다.

### 8-3. 장소 탭 흐름

장소 카드를 누르면 먼저 **장소 하단 시트**가 뜬다.

하단 시트에서 할 수 있는 것:

- 시간 추가
- 메모 추가
- 비용 추가

장소 하단 시트 상단의 장소 요약을 다시 누르면 **장소 상세 화면**으로 들어간다.

즉 장소는 두 단계로 접근한다.

- 1차: 일정 문맥 안에서 빠르게 조작하는 장소 시트
- 2차: 구글맵/장소 정보 기반 상세 화면

### 8-4. 메모 흐름

메모는 장소가 아니라, **day 타임라인 사이에 자유롭게 끼워 넣는 텍스트 아이템**이다.

- 장소처럼 번호가 붙지 않는다.
- 타임라인에서는 중립적인 작은 점으로 표시된다.
- 메모 추가를 누르면 메모 바텀시트가 뜬다.

메모 시트에서 할 수 있는 것:

- 메모 입력
- 메모 저장
- 시간 추가

### 8-5. 비용 추가 흐름

장소 하단 시트의 `비용 추가` 또는 캘린더의 `지출 추가`에서 진입한다.

입력 구조:

1. 통화 선택
2. 금액 입력
3. 날짜 선택
4. 결제수단 선택
5. 항목명
6. 상위 카테고리 선택
7. 장소 선택
8. 메모 및 사진
9. 저장

중요 정책:

- 지출은 `장소 또는 상위 카테고리`가 아니라 **`장소 + 상위 카테고리` 둘 다 선택 가능**해야 한다.
- 장소가 없는 지출은 장소 없이 상위 카테고리만 저장 가능하다.
- 모든 지출은 **현지 통화와 한국 원화 기준을 함께 볼 수 있어야 한다.**

### 8-6. 체크리스트 흐름

1. 사용자가 여행 안에서 체크리스트로 진입
2. 카테고리(섹션)별 준비물을 본다
3. 항목을 체크한다
4. `편집`을 누르면 편집 모드로 진입
5. 카테고리 추가, 아이템 추가, 삭제를 수행

핵심 원칙:

- 체크리스트는 flat list가 아니라 **카테고리별 섹션 리스트**다.
- `필수 준비물`, `기본 짐싸기` 같은 섹션은 고정이 아니다.
- 섹션도 사용자가 직접 추가/수정 가능해야 한다.
- 아이템 추가는 별도 입력 화면에서 한다.

### 8-7. OCR 일정 가져오기 흐름

1. 사용자는 여행을 생성하거나 선택한 뒤, `일정 편집 화면`의 `OCR 가져오기`로 진입한다.
2. OCR 검수 화면으로 이동한다.
3. 사진 촬영 또는 앨범 선택을 한다.
4. 추출된 일정 후보를 검수한다.
5. 검수 결과를 현재 선택된 여행의 일정 편집 화면에 반영한다.

핵심 원칙:

- OCR 결과는 일정과 구분된 특수 아이템처럼 남기지 않는다.
- 검수 화면에서만 정리하고, 일정 화면 안에서는 일반 일정 아이템처럼 보이게 한다.
- OCR은 반드시 `여행 생성 또는 여행 선택 이후`에만 시작할 수 있다.

### 8-8. 캘린더 흐름

#### 전역 캘린더

모든 여행의 지출을 날짜 기준으로 본다.

- 날짜를 누르면 해당 날짜의 전체 여행 지출 내역이 하단 시트에 보인다.
- 시트를 위로 올리면 선택 주간만 남기고 상세 내역에 집중할 수 있다.

#### 여행 전용 캘린더

선택된 여행 안에서만 보이는 지출 캘린더

- 해당 여행 기간과 여행명 메타를 보여준다.
- 날짜별 지출을 본다.
- `장소별 / 상위 카테고리별` 토글을 가진다.
- 현지 통화와 원화를 함께 보여준다.
- 하단 상세 영역에서는 날짜별 상세 내역과 집계 리스트를 함께 본다.

### 8-9. 날씨 흐름

1. 사용자가 여행 안에서 날씨 화면으로 진입
2. 여행 기간 중 해당 국가의 날씨를 본다
3. 실제 데이터는 대표 도시 기준으로 보여준다
4. 현지 시각과 시차도 함께 참고한다

### 8-10. 환율 흐름

1. 사용자가 홈의 `오늘 환율` 블록 또는 별도 환율 참고 화면으로 진입한다.
2. 선택 날짜 기준 환율을 본다.
3. 전 영업일 대비 증감값 / 증감률을 함께 본다.
4. 환전 또는 지출 기록 전 참고 정보로 사용한다.

## 9. 핵심 기능 명세

### 9-1. 여행

- 여행 생성
- 여행 정보 수정
- 여행 검색
- 현재 선택된 여행 변경
- 여행 목적지 선택 / 순서 변경
- 여행 기간 선택
- 여행 동행 유형 선택
- 여행 스타일 태그 선택
- 여행 상태 구분
  - 진행 예정
  - 진행 중
  - 완료

여행 상태 규칙:

- 상태는 사용자가 직접 바꾸지 않는다.
- 서버가 여행 시작일/종료일을 기준으로 `planned / active / completed`를 계산하고 저장한다.
- 여행 생성, 여행 날짜 수정, 로그인 시점, 일별 배치 또는 목록 조회 시점에 상태를 동기화한다.
- 여행명은 사용자가 직접 수정할 수 있지만, 최초 생성 시에는 선택 도시 기준으로 자동 생성한다.
- 여러 도시를 선택한 경우 첫 번째 도시를 대표 도시로 사용하고, 전체 도시는 별도 목적지 목록으로 관리한다.

### 9-2. 일정

- day별 타임라인
- 항공편 추가
- 장소 검색 후 삽입
- 장소 카드 드래그
- 다른 day로 이동
- 메모 추가
- 전면 지도 보기
- OCR 일정 가져오기
- PDF 공유

일정 화면 원칙:

- 지도 미리보기는 핵심이다.
- `day 일정`, `전체 지도` 같은 장식성 문구는 최소화한다.
- 장소 번호는 장소 카드 옆에만 표시한다.
- 메모는 숫자 없이 중립 점으로 표시한다.
- 항공편은 비행기 아이콘, 출발 시간, 출발/도착 공항 코드, 항공사/편명으로 표시한다.
- 공항은 공항 검색 결과에서 선택하며, 항공사는 추천 목록 선택 또는 직접 입력을 모두 허용한다.
- 구매 관련 광고 배너, 셀프패키지, 일행과 함께 일정짜기 기능은 MVP 일정 화면에서 제외한다.

### 9-3. 장소

- Google Places 자동완성 검색
- 일정에 즉시 삽입
- 장소 하단 시트
- 장소 상세 화면

장소 상세 정보 예시:

- 이름
- 위치
- 영업시간
- 현재 영업 여부
- 평점
- 저장 수 / 리뷰 수
- 대표 이미지
- 설명
- 대표 메뉴

### 9-4. 지출

- 날짜별 입력
- 장소 + 상위 카테고리 동시 선택
- 통화 선택
- 현지 통화 / 원화 동시 표시
- 사진 메모
- 전역 캘린더에서 전체 여행 지출 조회
- 여행 전용 캘린더에서 해당 여행 지출만 조회

중요 정책:

- `수입`은 메인 개념으로 다루지 않는다.
- 메인 UX는 `지출` 중심으로 간다.
- 환불/정산 같은 예외만 별도 성격으로 처리한다.

### 9-5. 상위 카테고리

카테고리 관리 정책:

- 상위 카테고리만 관리한다.
- 기본 카테고리는 `기타` 하나만 둔다.
- 나머지는 사용자가 직접 추가한다.
- `기본 / 사용자 정의`를 나누지 않는다.
- 장소와 상위 카테고리는 다른 개념이다.

### 9-6. 체크리스트

- 카테고리별 섹션 리스트
- 섹션 사용자 커스텀
- 아이템 추가
- 아이템 삭제
- 편집 모드
- 아이템별 더보기
- 삭제용 바텀시트

### 9-7. 날씨 / 환율

#### 날씨

- 진행 중 여행이 있을 경우 홈에 `오늘 날씨`
- 여행 안에서 `여행 기간 중 국가 날씨`
- 시차 / 현지 시각 표시

#### 환율

- 진행 중 여행이 있을 경우 홈에 `오늘 환율`
- 별도 `환율 참고` 화면 제공
- 전 영업일 대비 증감값 / 증감률 표시 가능
- 홈의 환율 블록을 누르면 환율 참고 화면으로 이동한다.

### 9-8. 설정 / 기본값

- 기본 통화
- PDF 기본 범위
- 동선 기준 이동수단
- 문의 / 로그아웃

## 10. 화면별 역할 정리

### 홈

`현재 무엇을 해야 하는지` 보여주는 화면

- 여행 목록 전체를 보여주는 화면이 아니다.
- 실행 대시보드 역할이다.

### 보관함

`내 여행을 찾고 선택하는 화면`

- 검색 버튼
- 상태 필터
- 작은 `+` 액션

### 일정

`현재 선택한 여행을 실제로 운영하는 화면`

- 동선
- 장소
- 메모
- 준비물 / 날씨 / 여행 캘린더로의 진입점

### 캘린더

`모든 여행 지출을 날짜 기준으로 보는 화면`

- 전역 지출 흐름
- 주간 전환 가능한 상세 시트

### 여행 전용 캘린더

`선택된 여행만의 지출 흐름을 보는 화면`

- 장소별 / 상위 카테고리별
- 현지 통화 / 원화 동시 표기

### 설정 / 마이페이지

`계정과 기본값을 관리하는 보조 화면`

- 기본 통화
- PDF 기본 범위
- 동선 기준 이동수단
- 문의 / 로그아웃

## 14. 최종 요약

Tripline은 단순한 여행 일정 앱도 아니고, 단순한 가계부 앱도 아니다.

Tripline은

- 여행을 만들고
- 그 여행 안에서
  - 일정
  - 준비물
  - 날씨
  - 지출

을 함께 관리하는 **여행 실행 앱**이다.

그리고 전역 캘린더를 통해서는 모든 여행의 지출을 날짜 기준으로 다시 모아볼 수 있다.

즉 Tripline의 핵심은  
**“여행을 만들고, 그 여행을 실제로 운영하게 해주는 것”**이다.
