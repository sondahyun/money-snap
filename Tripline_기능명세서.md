## 1. 문서 목적

이 문서는 Tripline의 화면별 기능, 사용자 흐름, 핵심 정책, 예외 처리 기준을 개발 관점에서 정리한 기능 명세서다.

Tripline은 `여행을 먼저 만들고`, `그 여행 안에서 일정, 준비물, 날씨, 지출을 운영하는 여행 실행 앱`이며, 캘린더 탭은 예외적으로 `모든 여행의 지출을 날짜 기준으로 모아보는 전역 화면`이다.

## 2. 서비스 범위

MVP 범위에 포함되는 핵심 기능은 아래와 같다.

- 로그인 / 회원가입
- 홈 대시보드
- 보관함
- 여행 생성 / 수정 / 검색
- 일정 화면 / 일정 편집
- 장소 검색 / 장소 상세 / 장소 하단 시트
- 메모 추가 / 수정
- OCR 일정 가져오기 / 검수
- 준비물 체크리스트 보기 / 편집 / 아이템 추가
- 전역 캘린더
- 여행 전용 캘린더
- 지출 입력 / 지출 상세
- 상위 카테고리 관리
- 날씨 화면
- 환율 참고 화면
- 설정 / 기본값 관리
- PDF 공유

## 3. 공통 용어

- `여행`: 사용자가 생성한 하나의 여행 단위. 모든 핵심 기능의 기준이 된다.
- `현재 선택된 여행`: 일정, 준비물, 날씨, 여행 전용 캘린더가 참조하는 대상 여행.
- `전역 캘린더`: 모든 여행의 지출을 날짜 기준으로 통합해 보여주는 화면.
- `여행 전용 캘린더`: 특정 여행의 지출만 보여주는 화면.
- `장소`: Google Places 기반으로 검색/추가되는 일정 아이템.
- `메모`: 장소가 아닌 자유 일정 아이템. 타임라인 중간에 삽입된다.
- `상위 카테고리`: 지출 분류용 사용자 정의 분류. 장소와는 별개 개념이다.
- `환불/정산`: 일반 수입이 아니라 예외적 플러스 흐름으로 기록되는 항목.

## 4. 공통 정책

### 4-1. 여행 선행 원칙

- 사용자는 원칙적으로 먼저 여행을 생성해야 한다.
- 여행 생성 직후 생성된 여행은 자동으로 `현재 선택된 여행`이 된다.
- 보관함 / 여행 검색 / 홈 요약에서 여행을 누르면 해당 여행이 `현재 선택된 여행`으로 갱신된다.
- 일정, 준비물, 날씨, 여행 전용 캘린더는 반드시 특정 여행에 종속된다.
- 여행이 없는 상태에서 일정 탭에 진입하면 `여행 생성` 또는 `보관함 이동`을 안내한다.
- OCR은 예외 없이 `여행 생성 또는 여행 선택 이후`에만 시작할 수 있다.

### 4-2. 지출 정책

- 메인 개념은 `수입`이 아니라 `지출`이다.
- 환불/정산은 예외적인 플러스 흐름으로 처리한다.
- 지출 1건은 `장소 + 상위 카테고리`를 동시에 가질 수 있다.
- 장소가 없는 지출은 허용하되, 상위 카테고리는 반드시 가진다.
- 모든 지출은 `현지 통화`와 `원화`를 함께 보관하고 표시한다.

### 4-3. 일정 정책

- 장소는 번호 배지가 붙는다.
- 메모는 번호 없이 중립 점으로 표시된다.
- 메모는 장소가 아니라 자유 일정 아이템이다.
- 장소 시트에서 `비용 추가`, `시간 추가`, `메모 추가`가 가능하다.
- 장소 하단 시트 상단의 요약 영역을 한 번 더 누르면 장소 상세 화면으로 진입한다.

### 4-4. 체크리스트 정책

- 체크리스트는 카테고리별 섹션 리스트다.
- 섹션은 사용자가 직접 추가/수정/삭제할 수 있다.
- 아이템 추가는 별도 입력 화면에서 수행한다.
- 보기 화면에서는 체크가 중심이고, 편집은 별도 편집 모드에서 한다.

### 4-5. 캘린더 정책

- 전역 캘린더는 `모든 여행`의 지출을 보여준다.
- 여행 전용 캘린더는 `현재 선택된 여행`의 지출만 보여준다.
- 캘린더 하단 시트는 위아래로 조정 가능해야 한다.
- 하단 시트를 위로 올리면 월 전체 대신 `선택 주간만` 보이도록 전환한다.

### 4-6. 삭제 정책

- 여행 삭제는 hard delete가 아니라 soft delete를 기본으로 한다.
- 일정 아이템, 준비물 섹션/아이템, 지출, 지출 사진, 상위 카테고리는 사용자 삭제 시 soft delete로 처리한다.
- 일정 일차, 숙소 예약 그룹, 동선 구간은 일정 편집 과정에서 내부적으로 soft delete될 수 있으며, 삭제된 데이터는 화면에 다시 노출하지 않는다.
- OCR 후보는 삭제보다 `승인 / 거절 / 수정` 상태 관리가 우선이며, 별도 삭제 기능은 MVP 필수 범위로 두지 않는다.

## 6. 상세 기능 명세

### F-01. 인증

#### 목적

- 사용자 계정을 생성하고 로그인 상태를 유지한다.

#### 세부 기능

- 이메일 회원가입
- 이메일 로그인
- 토큰 재발급
- 로그아웃

#### 입력

- 이메일
- 비밀번호
- 닉네임(회원가입 시)

#### 처리 규칙

- 비밀번호는 서버 저장 시 단방향 해시로 처리한다.
- 로그인 성공 시 Access Token / Refresh Token을 발급한다.

#### 예외

- 이메일 중복
- 비밀번호 불일치
- 비활성 사용자

### F-02. 여행 생성 / 수정 / 검색

#### 목적

- 사용자가 Tripline 안에서 관리할 여행 단위를 만든다.

#### 세부 기능

- 여행 생성
- 여행 수정
- 여행 검색
- 여행 도시 선택
- 선택 도시 순서 변경
- 여행 기간 선택
- 여행 동행 유형 선택
- 여행 스타일 태그 선택
- 상태 필터
  - 진행 예정
  - 진행 중
  - 완료

#### 입력

- 목적지 목록
- 목적지 순서
- 여행 시작일 / 종료일
- 기준 통화
- 동행 유형(선택)
- 여행 스타일 태그(선택)
- 여행명(자동 생성 후 수정 가능)

#### 처리 규칙

- 예산은 핵심 입력에서 제외한다.
- 여행 생성은 `도시 선택 -> 순서 확인/변경 -> 기간 선택 -> 스타일 선택` 순서로 진행한다.
- 여러 도시를 선택할 수 있으며, 선택한 도시의 순서를 저장한다.
- 첫 번째 선택 도시는 대표 도시로 저장한다.
- 여행명은 선택 도시 기준으로 자동 생성한다.
  예: `후쿠오카, 도쿄, 오사카 여행`
- 동행 유형과 여행 스타일 태그는 선택 입력이며, 사용자는 `다음에 하기`로 건너뛸 수 있다.
- 여행 생성 시 날짜 범위에 맞는 `day`가 자동 생성된다.
- 여행 생성 직후 해당 여행은 사용자의 `현재 선택된 여행`으로 저장된다.
- 여행 상태는 사용자가 직접 수정하지 않는다.
- 여행 상태는 서버가 여행 날짜 기준으로 `planned / active / completed`를 계산해 저장한다.
- 상태 동기화 시점은 여행 생성, 여행 날짜 수정, 로그인 직후, 일 1회 배치, 여행 목록/상세 조회 시점이다.

#### 예외

- 시작일 > 종료일
- 필수값 누락

### F-03. 홈

#### 목적

- 현재 해야 할 일을 가장 빠르게 보여주는 실행 대시보드

#### 상태별 동작

##### 진행 중 여행이 있을 때

- 현재 여행 요약
- 다음 일정
- 오늘 동선 요약
- 오늘 날씨
- 오늘 환율
- 오늘 지출 / 누적 지출
- 최근 기록

##### 진행 중 여행은 없지만 현재 선택된 여행이 있을 때

- 현재 선택된 여행 요약
- 일정 보기
- 준비물 보기
- 날씨 보기
- 여행 전용 캘린더 보기

##### 여행이 아예 없을 때

- 새 여행 만들기

#### 표시 규칙

- 여행 목록 전체는 홈에서 보여주지 않는다.
- `진행 중 여행`이 있으면 실행형 대시보드를 우선 노출한다.
- `진행 중 여행`이 없더라도 `현재 선택된 여행`이 있으면 해당 여행 진입용 요약 블록을 노출한다.
- 여행이 하나도 없을 때만 시작용 빈 상태를 노출한다.
- 홈의 `오늘 환율` 블록을 누르면 별도 환율 참고 화면으로 이동한다.

### F-05. 일정 화면

#### 목적

- 현재 선택된 여행을 실제로 운영하는 핵심 화면

#### 세부 기능

- 상단 여행 정보 표시
- 지도 미리보기
- day별 타임라인
- 장소 카드
- 메모 카드
- 항공편 카드
- 숙소 카드
- 거리 / 도보 / 차량 정보
- day별 `항공편 추가`
- day별 `숙소 추가`
- day별 `장소 추가`
- day별 `메모 추가`
- 여행 가계부 진입
- 일정 아이템 삭제
- 일정 편집 진입
- 전면 지도 보기

#### 세부 규칙

- `day 일정`, `전체 지도` 같은 장식성 문구는 최소화한다.
- 지도를 탭하면 전면 지도로 이동한다.
- 장소 번호는 장소 카드 옆에만 표시한다.
- 메모는 중립 점으로만 표시한다.
- 항공편은 비행기 아이콘과 출발 시간, `출발공항 출발시간 - 도착공항 도착시간` 형식으로 표시한다.
- 항공편 추가는 `출발일 선택 -> 항공사 선택/입력 -> 편명 입력 -> 출발/도착 공항 검색 -> 출발/도착 일시 선택` 순서로 진행한다.
- 숙소 추가는 `숙소 검색 -> 숙소 선택 -> 체크인/체크아웃 날짜 선택` 순서로 진행한다.
- 숙소 날짜는 현재 여행 일정 안에서만 선택할 수 있다.
- 숙소 저장 시 체크인일부터 체크아웃 전날까지 각 day에 동일 숙소 카드가 표시된다.
- 여러 day에 표시된 숙소 카드는 하나의 숙소 예약 단위로 묶어 수정 / 삭제한다.
- 일정 화면의 `가계부` 버튼을 누르면 현재 여행 전용 가계부로 이동하고, day별 `비용 추가`에서 지출 입력으로 진입한다.
- `편집` 버튼은 day 1 헤더에만 노출한다.
- 구매 관련 광고 배너, 셀프패키지, 일행과 함께 일정짜기 기능은 MVP 일정 화면에서 제외한다.

#### 빈 상태

- 여행이 없으면 `여행 생성` / `보관함 보기` 안내
- 여행은 있으나 일정이 없으면 `장소 추가` / `OCR 가져오기` 중심 안내

#### 삭제 규칙

- 일정 아이템 삭제는 hard delete가 아니라 soft delete다.
- 삭제된 일정 아이템은 day 타임라인과 전면 지도에서 제외한다.
- 동선 구간은 일정 아이템 삭제 또는 재정렬 결과에 따라 함께 숨기거나 재계산한다.

### F-06. 장소

#### 목적

- Google Places 기반 장소를 검색하고 일정에 삽입한다.

#### 세부 기능

- 자동완성 검색
- 카테고리 필터(맛집 / 관광 / 숙소)
- 지도 기반 추천 리스트
- 일정에 즉시 추가
- 장소 하단 시트
- 장소 상세 진입

#### 장소 하단 시트에서 가능한 것

- 시간 추가
- 메모 추가
- 비용 추가
- 리뷰쓰기(후속/프로토타입)

#### 장소 상세에서 보이는 것

- 이름
- 원문명
- 위치
- 평점 / 저장 수 / 리뷰 수
- 예약 가능 여부
- 영업시간
- 대표 이미지
- 설명
- 대표 메뉴

---

### F-07. 메모

#### 목적

- 장소가 아닌 자유 일정 아이템을 day 타임라인 안에 삽입한다.

#### 세부 기능

- 메모 추가
- 메모 수정
- 메모 시간 추가

#### 표시 규칙

- 장소 카드처럼 보이지 않되, 타임라인 흐름 안에는 포함된다.
- 번호는 붙지 않는다.
- 메모 카드를 누르면 메모 바텀시트가 열린다.

---

### F-08. OCR 일정 가져오기

#### 목적

- 사진 속 여행 일정표를 읽어 일정 후보로 변환한다.

#### 세부 기능

- 일정 편집에서 `OCR 가져오기`
- 카메라 촬영
- 앨범 선택
- OCR 텍스트 추출
- 후보 검수
- 일정 편집으로 반영

#### 처리 규칙

- OCR 업로드는 반드시 여행 생성 후, 해당 여행이 선택 또는 진입된 상태에서만 시작할 수 있다.
- 검수 완료 후에는 해당 여행의 일정 편집 화면에 반영한다.
- OCR 결과는 일정 화면에서 특수 상태로 남지 않는다.
- 검수 완료 후에는 일반 일정 아이템처럼 보인다.
- OCR 후보는 삭제보다 `승인 / 거절 / 수정` 상태 전이로 관리한다.

#### 예외

- 권한 거부
- OCR 결과 없음
- 일정 후보 0건

---

### F-09. 체크리스트

#### 목적

- 여행별 준비물을 카테고리(섹션)별로 관리한다.

#### 세부 기능

- 체크리스트 보기
- 체크 상태 변경
- 편집 진입
- 섹션 추가 / 수정 / 삭제
- 아이템 추가 / 수정 / 삭제
- 아이템 더보기 / 삭제 바텀시트

#### 구조 규칙

- 섹션 기반 리스트
- 섹션명 커스텀 가능
- 아이템 추가는 별도 입력 화면에서 처리
- 보기 화면과 편집 화면을 분리

#### 삭제 규칙

- 섹션 삭제는 soft delete다.
- 섹션이 삭제되면 하위 아이템도 함께 soft delete된다.
- 아이템 삭제도 soft delete이며, 삭제된 항목은 체크리스트 조회 결과에서 제외한다.

---

### F-10. 지출 입력 / 지출 상세

#### 목적

- 여행 중 실제 지출을 기록하고, 나중에 장소/카테고리/날짜 기준으로 다시 본다.

#### 지출 입력 필수 항목

- 통화
- 금액
- 날짜
- 결제수단
- 항목명
- 상위 카테고리

#### 진입 경로

- 일정 화면의 `가계부`
- 여행 전용 가계부의 `비용 추가`
- 장소 하단 시트의 `비용 추가`
- 전역 캘린더의 `지출 추가`

#### 선택 항목

- 장소
- 메모
- 사진

#### 지출 상세에서 가능한 것

- 세부 정보 확인
- 수정
- 삭제

#### 처리 규칙

- 모든 지출 응답은 `현지 통화 금액`, `원화 금액`, `적용 환율`을 함께 포함해야 한다.
- 현지 통화와 원화를 같이 저장한다.
- 환율은 저장 시점 기준 값을 사용한다.
- 환불/정산은 예외 타입으로 저장 가능하다.
- 지출 삭제는 soft delete다.
- 지출 삭제 시 연결된 사진도 함께 soft delete된다.

---

### F-11. 상위 카테고리 관리

#### 목적

- 지출 분류용 상위 카테고리를 관리한다.

#### 세부 기능

- 목록 조회
- 생성
- 수정
- 삭제

#### 규칙

- 장소와는 무관한 별도 관리 화면이다.
- 기본 카테고리는 `기타` 하나만 제공한다.
- 나머지는 사용자가 직접 추가한다.
- `기본 / 사용자 정의`를 별도 섹션으로 나누지 않는다.
- 카테고리 삭제는 soft delete다.
- 삭제된 카테고리는 신규 선택 목록에서 제외하지만, 과거 지출 기록의 참조는 유지한다.

---

### F-12. 전역 캘린더

#### 목적

- 모든 여행의 지출을 날짜 기준으로 한 번에 본다.

#### 세부 기능

- 월간 캘린더
- 월 지출 합계
- 날짜별 지출 표시
- 하단 상세 시트

#### 하단 시트 규칙

- 접힌 상태: 월 전체 + 요약
- 펼친 상태: 선택된 `주간만` 보이고, 아래는 세부 지출 내역 중심

#### 세부 지출 내역 규칙

- `+0`, `-0`은 표시하지 않는다.
- 실제 지출/환불이 있는 날짜만 숫자를 표시한다.
- 기본 집계 대상은 `지출`이며, 일반 수입은 전역 캘린더 범위에 포함하지 않는다.

---

### F-13. 여행 전용 캘린더

#### 목적

- 선택한 여행의 지출만 날짜 기준으로 본다.

#### 세부 기능

- 여행 메타(여행명, 국가/도시, 기간)
- 날짜별 지출 표시
- `장소별 / 상위 카테고리별` 토글
- 현지 통화 / 원화 동시 표시

#### 규칙

- 전역 캘린더와 별도 개념이다.
- 선택된 여행의 데이터만 포함한다.
- 하단 상세 영역에서는 날짜별 내역과 함께 `장소별 / 상위 카테고리별` 집계를 전환해 볼 수 있다.

---

### F-14. 날씨

#### 목적

- 여행 운영에 필요한 날씨와 시차 정보를 보여준다.

#### 세부 기능

- 홈의 오늘 날씨
- 여행 안에서 보는 여행 기간 날씨
- 현지 시각 / 시차 표시

#### 규칙

- 국가 기준으로 보여주되, 실제 데이터는 대표 도시 기준으로 사용한다.

---

### F-15. 환율

#### 목적

- 여행 중 환전/지출 판단을 위한 참고 환율을 제공한다.

#### 세부 기능

- 홈의 오늘 환율
- 별도 환율 참고 화면
- 전 영업일 대비 증감값 / 증감률 표시

#### 규칙

- API가 직접 전일 대비를 주지 않으면 서버에서 이전 영업일 데이터와 비교 계산한다.
- 홈의 `오늘 환율` 블록은 환율 참고 화면 진입점 역할을 한다.

---

### F-16. PDF 공유

#### 목적

- 선택한 여행 일정 또는 일부 범위를 PDF로 공유한다.

#### 세부 기능

- 전체 일정 PDF
- 특정 day PDF
- 오늘 일정 PDF

#### 규칙

- 지출은 PDF 기본 범위에서 제외한다.

---

### F-17. 설정 / 기본값 관리

#### 목적

- 사용자의 기본값과 앱 사용 기준을 관리한다.

#### 세부 기능

- 기본 통화 설정
- PDF 기본 범위 설정
- 동선 기준 이동수단 설정
- 문의 진입
- 로그아웃

#### 규칙

- 설정 화면은 여행 내부 기능이 아니라 사용자 계정 기준 보조 화면이다.
- 기본 통화, PDF 기본 범위, 동선 기준 이동수단은 사용자 설정값으로 저장한다.

## 7. 예외 및 빈 상태

### 7-1. 공통 빈 상태

- 여행 없음
- 진행 중 여행 없음
- 일정 없음
- 장소 검색 결과 없음
- OCR 결과 없음
- 체크리스트 항목 없음
- 해당 날짜 지출 없음

### 7-2. 공통 오류 상태

- 네트워크 오류
- 인증 만료
- 외부 API 호출 실패
- 업로드 실패

## 8. 외부 연동

- Google Places: 장소 검색 / 상세
- Google Maps: 지도 / 마커 / 동선 미리보기
- Routes API: 거리 / 도보 / 차량 시간
- 글로벌 날씨 API: 여행 기간 날씨 / 현지 시각
- 환율 API: 날짜별 환율 / 전 영업일 비교
- OCR 엔진: 일정표 인식

## 9. 후순위 / 제외

아래 기능은 현재 MVP 핵심 범위에서 제외하거나 후순위로 둔다.

- 공동 편집
- 공개 공유 링크
- 커뮤니티 / 피드
- 일반 여행 추천 피드
- 카드 문자 완전 자동 파싱
- 고도화된 AI 추천 피드

## 10. 최종 요약

Tripline의 기능 구조는 다음 한 문장으로 정리된다.

**사용자가 여행을 먼저 만들고, 그 여행 안에서 일정, 준비물, 날씨, 지출을 함께 운영하며, 전역 캘린더에서는 모든 여행의 지출을 날짜 기준으로 다시 모아보는 구조다.**
