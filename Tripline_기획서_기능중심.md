## 1. 서비스 소개

여행을 준비할 때는 일정은 메모에, 장소는 지도 앱에, 준비물은 체크리스트에, 지출은 가계부에 따로 적게 되는 경우가 많다.  
Tripline은 이렇게 흩어지는 여행 정보를 하나의 `여행` 단위로 묶어, 준비부터 이동, 기록까지 한 흐름으로 관리할 수 있게 만드는 서비스다.

핵심은 복잡한 추천보다 `지금 이 여행을 실제로 운영하는 경험`에 있다.

- 여행을 만든다.
- 그 여행 안에서 일정을 관리한다.
- 준비물을 체크한다.
- 날씨와 환율을 참고한다.
- 지출을 기록하고 다시 본다.

즉 Tripline은 여행을 보기 위한 앱이 아니라, **여행을 준비하고 움직이고 기록하기 위한 실행형 앱**이다.

## 2. 서비스 정의

**Tripline은 사용자가 여행을 먼저 만들고, 그 여행 안에서 일정, 준비물, 날씨, 환율, 지출을 함께 운영하는 여행 실행 앱이다.**

Tripline은 여행 추천이나 콘텐츠 소비보다, 여행을 실제로 준비하고 움직이고 기록하는 과정에 초점을 둔다.

## 3. 서비스 목표

Tripline이 해결하려는 핵심 문제는 아래와 같다.

- 장소, 일정, 준비물, 지출 정보가 여러 앱에 흩어진다.
- 여행 중에는 오늘 일정과 오늘 지출을 한 화면 맥락에서 보기 어렵다.
- 여행이 여러 개일 때 날짜별 소비 흐름을 한 번에 파악하기 어렵다.
- 여행 준비 단계와 여행 실행 단계가 분리되어 정보 흐름이 끊긴다.

Tripline은 이를 `여행` 단위로 묶는다.

- 여행을 만든다.
- 그 여행 안에서 day별 일정과 동선을 관리한다.
- 같은 여행 맥락에서 준비물을 관리한다.
- 같은 여행 맥락에서 날씨, 환율, 시차를 참고한다.
- 같은 여행 맥락에서 지출을 기록한다.
- 전역 캘린더에서 모든 여행의 지출을 날짜 기준으로 다시 본다.

## 4. 핵심 정책

### 4-1. 여행 선행 정책

- Tripline의 주요 기능은 기본적으로 `여행 생성` 이후 사용한다.
- 여행 생성 직후 해당 여행은 `현재 선택된 여행`이 된다.
- 일정, 준비물, 날씨, 여행 전용 캘린더는 모두 `현재 선택된 여행` 기준으로 동작한다.
- 전역 캘린더만 예외적으로 모든 여행의 지출을 통합해서 보여준다.
- OCR도 예외 없이 `여행 생성 또는 여행 선택 이후`에만 시작할 수 있다.

### 4-2. 지출 정책

- 메인 개념은 `수입`이 아니라 `지출`이다.
- 환불/정산은 예외적인 플러스 흐름으로 처리한다.
- 지출은 `장소 + 상위 카테고리`를 함께 가질 수 있다.
- 장소가 없는 지출도 허용하지만, 상위 카테고리는 반드시 가진다.
- 모든 지출은 현지 통화와 원화 기준을 함께 보여준다.

### 4-3. 일정 정책

- 장소는 day별 타임라인 안에 배치된다.
- 메모는 장소가 아니라 타임라인 중간에 삽입되는 자유 일정 아이템이다.
- 장소는 번호 배지를 가지며, 메모는 번호 없이 중립 점으로 표시한다.
- 장소 하단 시트에서 시간, 메모, 비용 추가를 빠르게 처리한다.

### 4-4. OCR 정책

- OCR은 여행을 생성하거나 선택한 이후, `일정 편집` 안에서만 시작한다.
- OCR은 현재 선택된 여행을 기준으로 동작하며, 검수 완료 후 해당 여행 일정에 반영한다.
- OCR 결과는 일정 화면에서 특수 상태로 남지 않고 일반 일정 아이템처럼 보인다.

### 4-5. 삭제 정책

- 여행, 일정, 준비물, 지출, 카테고리는 기본적으로 soft delete를 사용한다.
- 삭제된 데이터는 기본 조회에서 제외한다.
- OCR 후보는 삭제보다 `승인 / 거절 / 수정` 상태 관리가 우선이다.

## 5. 핵심 기능 구성

Tripline의 MVP 기능은 아래 6개 축으로 정리할 수 있다.

1. 여행 관리
2. 일정 운영
3. 준비물 관리
4. 지출 관리
5. 여행 참고 정보
6. 공통 관리 기능

## 6. 기능별 기획

### 6-1. 여행 관리

#### 목적

- 사용자가 여행 단위를 만들고 선택하며, 이후 모든 기능의 기준점을 만든다.

#### 세부 기능

- 회원가입 / 로그인
- 여행 생성
- 여행 수정
- 여행 검색
- 여행 도시 선택
- 선택 도시 순서 변경
- 여행 기간 선택
- 여행 동행 유형 선택
- 여행 스타일 태그 선택
- 여행 상태 조회
- 현재 선택된 여행 변경
- 보관함에서 여행 목록 조회

#### 핵심 동작

- 사용자는 여행 생성 시 먼저 `해외도시 / 국내도시` 탭에서 목적지를 선택한다.
- 여러 도시를 선택할 수 있으며, 선택한 도시의 순서를 변경할 수 있다.
- 여행 시작일과 종료일을 선택한다.
- `누구와`, `여행 스타일`은 선택 입력이며, 건너뛸 수 있다.
- 여행명은 첫 도시 기준으로 자동 생성한다.
  예: `후쿠오카, 도쿄, 오사카 여행`
- 여행 생성 직후 해당 여행이 현재 선택된 여행이 된다.
- 보관함에서는 진행 예정 / 진행 중 / 완료 필터와 검색을 통해 여행을 찾는다.
- 사용자가 여행을 선택하면 일정, 준비물, 날씨, 여행 전용 캘린더가 그 여행 기준으로 바뀐다.

#### 중요 원칙

- 예산은 MVP 핵심 입력에서 제외한다.
- 여행 상태는 사용자가 직접 바꾸지 않고 서버가 날짜 기준으로 계산한다.
- 첫 번째 선택 도시는 대표 도시로 사용하고, 전체 선택 도시는 목적지 목록으로 저장한다.
- 동행 유형과 여행 스타일은 추천/표시용 보조 정보이며 필수 입력이 아니다.

### 6-2. 일정 운영

#### 목적

- 여행의 핵심 경험인 day별 일정과 동선을 실제로 관리한다.

#### 세부 기능

- day별 타임라인 조회
- 장소 검색 및 삽입
- 항공편 추가
- 숙소 검색 및 날짜 범위 추가
- 장소 순서 변경
- 다른 day로 이동
- 여행 가계부 진입
- 메모 추가 / 수정
- 장소 하단 시트
- 장소 상세 화면
- 전면 지도 보기
- OCR 일정 가져오기
- PDF 공유

#### 핵심 동작

- 사용자는 선택된 여행의 일정 화면에서 day별 장소를 배치한다.
- 항공편은 출발일을 먼저 선택한 뒤 항공사/편명, 출발/도착 공항, 출발/도착 일시를 입력해 추가한다.
- 항공사는 추천 목록에서 선택하거나 직접 입력할 수 있고, 공항은 공항 검색 결과에서 선택한다.
- 숙소는 검색 결과에서 선택한 뒤 체크인/체크아웃 날짜를 여행 기간 안에서 고르고, 선택한 날짜 범위의 각 day에 카드로 표시한다.
- 장소 검색 결과를 바로 일정에 넣는다.
- 일정 화면의 `가계부` 버튼으로 현재 여행 전용 지출 화면에 진입하고, day별 비용 추가를 시작한다.
- 드래그로 순서를 바꾸거나 다른 day로 옮긴다.
- 지도 미리보기를 눌러 전면 동선 지도로 이동한다.
- 장소 카드를 누르면 먼저 장소 하단 시트가 열리고, 요약 영역을 다시 누르면 장소 상세 화면으로 이동한다.
- 메모는 장소 사이에 자유롭게 삽입한다.

#### 중요 원칙

- 장소는 번호 배지 중심으로, 메모는 중립적 요소로 구분한다.
- 항공편은 비행기 아이콘과 공항 코드, 항공사/편명으로 장소/메모와 구분한다.
- 숙소는 숙박 기간에 걸쳐 반복 표시되지만, 하나의 숙소 예약 맥락으로 이해되게 표현한다.
- 지도 미리보기는 장식이 아니라 핵심 기능이다.
- 일정이 없는 상태에서는 `장소 추가`와 `OCR 가져오기`를 중심으로 안내한다.
- 구매 관련 광고 배너, 셀프패키지, 일행과 함께 일정짜기 기능은 MVP 일정 화면에서 제외한다.

### 6-3. 준비물 관리

#### 목적

- 여행별 준비물을 카테고리 단위로 구조화해 체크하고 편집할 수 있게 한다.

#### 세부 기능

- 체크리스트 보기
- 체크 상태 변경
- 섹션 추가 / 수정 / 삭제
- 아이템 추가 / 수정 / 삭제
- 편집 모드

#### 핵심 동작

- 사용자는 여행 안에서 체크리스트 화면으로 진입한다.
- 섹션별 준비물을 본다.
- 필요한 항목을 체크한다.
- 편집 모드에서 섹션과 아이템을 추가하거나 삭제한다.

#### 중요 원칙

- 체크리스트는 flat list가 아니라 섹션 기반 구조다.
- 기본 섹션을 강제하지 않는다.
- 아이템 추가는 별도 입력 화면에서 처리한다.

### 6-4. 지출 관리

#### 목적

- 여행 중 실제 사용한 비용을 기록하고, 날짜 / 장소 / 카테고리 기준으로 다시 볼 수 있게 한다.

#### 세부 기능

- 지출 입력
- 지출 상세 조회
- 지출 수정 / 삭제
- 상위 카테고리 관리
- 지출 사진 / 메모 첨부
- 전역 캘린더
- 여행 전용 캘린더

#### 핵심 동작

- 일정 화면의 `가계부`, 장소 하단 시트 또는 캘린더에서 지출 입력으로 진입한다.
- 통화, 금액, 날짜, 결제수단, 항목명, 상위 카테고리를 입력한다.
- 필요하면 장소, 메모, 사진을 추가한다.
- 저장된 지출은 현지 통화와 원화 기준으로 함께 보여준다.
- 전역 캘린더에서는 모든 여행의 지출을 날짜 기준으로 본다.
- 여행 전용 캘린더에서는 선택된 여행의 지출만 본다.
- 여행 전용 캘린더에서는 `장소별 / 상위 카테고리별` 집계를 전환해 볼 수 있다.

#### 중요 원칙

- Tripline의 지출 UX는 `가계부`보다 `여행 문맥 기록`에 가깝다.
- 전역 캘린더는 모든 여행의 흐름을, 여행 전용 캘린더는 특정 여행의 소비 흐름을 보여준다.
- 기본 집계는 지출 중심이며 일반 수입은 포함하지 않는다.

### 6-5. 여행 참고 정보

#### 목적

- 여행 실행에 필요한 참고 정보를 같은 맥락에서 제공한다.

#### 세부 기능

- 오늘 날씨
- 여행 기간 날씨
- 현지 시각 / 시차
- 오늘 환율
- 환율 참고 화면

#### 핵심 동작

- 홈에서는 진행 중 여행이 있을 때 오늘 날씨와 오늘 환율을 노출한다.
- 여행 안에서는 여행 기간 중 국가 날씨를 대표 도시 기준으로 보여준다.
- 환율 참고 화면에서는 선택 날짜 기준 환율과 전 영업일 대비 변동값을 보여준다.

#### 중요 원칙

- 날씨와 환율은 메인 기능이 아니라 `여행 운영 보조 정보`다.
- 하지만 지출 기록 전 참고 맥락으로 접근성이 좋아야 한다.

### 6-6. 공통 관리 기능

#### 목적

- 사용자 계정과 기본값을 관리해 반복 입력을 줄인다.

#### 세부 기능

- 기본 통화 설정
- PDF 기본 범위 설정
- 동선 기준 이동수단 설정
- 문의
- 로그아웃

#### 핵심 동작

- 설정 화면에서 사용자 기본값을 수정한다.
- 기본 통화는 여행 생성과 지출 입력의 기본값에 영향을 준다.
- PDF 기본 범위와 동선 기준 이동수단은 이후 관련 기능의 기본 선택값으로 사용된다.

## 7. 핵심 화면과 기능 연결

### 홈

- 현재 해야 할 일을 보여주는 실행 대시보드
- 현재 여행 요약, 오늘 일정, 날씨, 환율, 지출 진입
- 여행이 없으면 여행 생성 진입

### 보관함

- 여행을 검색, 필터링, 선택하는 허브

### 일정

- 장소, 메모, 동선, OCR, PDF 공유까지 연결되는 핵심 운영 화면

### 캘린더

- 모든 여행 지출을 날짜 기준으로 보는 전역 화면

### 여행 전용 캘린더

- 특정 여행의 지출 흐름을 장소/카테고리 단위로 보는 화면

### 설정

## 10. 최종 요약

Tripline은 단순한 일정 앱이 아니라, **여행을 만들고 그 여행을 실제로 운영하게 해주는 기능 중심 앱**이다.

핵심은 아래 두 줄로 정리된다.

- 여행을 먼저 만들고,
- 그 여행 안에서 일정, 준비물, 날씨, 환율, 지출을 함께 관리한다.

그리고 전역 캘린더를 통해서는 모든 여행의 지출을 날짜 기준으로 다시 모아볼 수 있다.
